先编译再解释

JS 的代码运行分为两个阶段，编译和执行，代码会在 编译后再执行，变量和函数的声明以及确定作用域的过程都是发生在编译阶段。

总结

  • var bar = 1 会被才分为 var bar; 和 bar = 1，声明会在编辑阶段生效，赋值会在解释阶段有效。
  • var 声明和函数声明在 JS 执行中被率先处理，这个过程被叫做提升。
  • 在提升中，函数优先。
  • 如果你觉得本节内容较难，你可以把同一作用域内的所有变量声明和函数声明放到程序的最前，函数放到变量后面，这样就是程序执行的顺序。
