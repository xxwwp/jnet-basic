基础布局

行级元素的布局

水平排版

行级元素的水平排版使用 text-align 属性，它有以下值：

值 描述
left 左侧对其
center 居中对其
right 右侧对其
justify 两端对齐，最后一行靠左

垂直排版

垂直排版有两个属性：

  • line-height：用来设置字体的行高
  • vertical：用来设置垂直方向上的排列

line-height 属性有以下值：

值 描述
normal 默认值，记住不是 auto
数值 像素，百分比等

vertical-align 用来设置一个行内元素在其所在行的垂直位置，并不是定义元素内容的垂直排列，所以 vertical-align 应用在子元素上。它有以下值：

值 描述
baseline 默认值，基于文字基线对齐
middle 基于中线对齐
top 基于顶部对齐
bottom 基于底部对齐
text-top 基于字体顶部对齐
text-bottom 基于字体底部对齐

留白

设置 white-space 属性，可以改变渲染时对留白的处理，留白由缩进、空格、换行等组成，它有以下值：

值 描述
normal 默认值，忽略留白
pre 原样渲染（不会自动换行，pre 元素的默认样式）
nowrap 文本永不换行，除非插入 br 元素
pre-wrap 自动换行并保留空白
pre-line 自动换行并合并空白

溢出

文本溢出分为两种情况：

  • 纵向溢出：父级设置了高度，文本换行后溢出
  • 横向溢出：文本设置了 white-space:nowrap，超出父级宽度溢出

如果父级设置了 overflow:hidden，那么 纵向溢出的文本只能被剪裁，横向溢出的文本可以通过 text-overflow 设置，text-overflow 有以下值：

值 描述
cilp 默认值，溢出部分被剪裁
ellipsis 溢出显示为省略号

定位布局

通过设置元素的 position 样式，可以改变其在文档中的定位方式，默认情况下，元素的位置根据文档流的排列放置，这种定位方式被称作静态定位。

通过设置 position 的值，可以有以下几种定位方式：

值 描述 备注
static 静态定位 默认定位
relative 相对定位
absolute 绝对定位
fixed 固定定位
sticky 粘性定位 兼容性不是非常好

相对定位与偏移

相对定位占据文档流中的位置，不过可以通过设置偏移量来改变它的最终位置。

用来设置偏移量的属性有 4 个：分别是 top、bottom、left、right，它们标记了定位元素的盒子边界到其定位参考位置的偏移量。

定位参考的位置：元素在文档流中原本的位置。

定位参考的位置：最近的 position 不是 static 的祖先，如果没有，则参考的是首屏的位置。

绝对定位元素会脱离当前的文档流。

层叠

定位元素之间有可能会因为偏移层叠在一起，通过 z-index 属性可以设置它们之间如何相互覆盖，z-index 属性又称 层级，层级对静态定位的元素无效。

z-index 属性有以下值：

值 描述
auto 默认值
整数值 值最大的定位元素显示到最前，按 z-index 大小堆叠

层叠上下文的产生条件如下：

  • position 值为 absolute 或 relative 且 z-index 值不为 auto 的元素。
  • position 值为 fixed 或 sticky 的元素。

浮动布局

浮动布局最初被用于在文字排版中，让文字可以在图片周围环绕，但是后来浮动布局被开发者用在各式各样的地方。

可以给元素设置 float 属性来让元素浮动，其他元素会环绕这个浮动元素来布局。

值 描述
none 不浮动
left 使元素浮动在左侧
right 使元素浮动在右侧

浮动元素的 display 属性会自动变为 block。

行内浮动

一个元素如果浮动起来，那么它周围的行内级元素或者文本就会怀绕在这个元素周围排列。

  • 浮动元素周围的文本会环绕这个元素排列

  • 浮动元素只能撑起 BFC 环境的高度，并不能撑起父级的高度，需要让父级包裹住浮动元素，需要创建新的 BFC 区域，否则浮动元素会产生溢出

  • 浮动元素溢出的部分也会影响其他行内级元素

块内浮动

  • 同一个 BFC 内的块元素在布局上会无视浮动元素
  • 同一个 BFC 内的浮动元素总是在块元素之上，所以浮动元素的样式会覆盖在块元素上

浮动元素之间的影响

  • 浮动方向相同的元素会按照浮动方向依次排列
  • 溢出的浮动元素会向下排列，并不是换行

消除浮动元素带来的影响

  • 清除浮动 clear

    值 描述
    none 不清除浮动 默认
    left 清除左侧的浮动
    left 清除右侧的浮动
    both 清除两侧的浮动

    清除浮动会消除浮动带来的影响，元素会从新的行开始布局

    • 从子元素上清理
    • 从父元素上清理 使用伪元素

  • 创建新的 BFC 环境（只能清除部分影响，比如块级元素还是会靠着浮动元素排列，只是不会在浮动元素的下方）

    • overflow 不等于 visible
    • display 等于 inline-block
    • float 不等于 none
    • 绝对定位和固定定位
    • display: flow-root 兼容性很差，但是无副作用

浮动元素在其他地方的应用

  • 水平方向上的列表（导航栏、图片）
  • 左中右布局
