盒模型属性并不全都有效，盒模型也有分类，部分盒子可能无法设置外边距或者高度。

盒模型分类

元素的 display 属性决定了元素渲染时的盒模型，根据盒模型的大致风格，CSS 把元素分为以下几类：

  • 块级元素：HTML 中的块级元素的盒模型，比如：p、div、table、ul、li 等等。

  • 行内级元素：HTML 中的内联元素的盒模型。

    • 行内块元素：默认只有 img 元素。
    • 行内元素：各种内联元素比如：span、em、del 等等。

  • 表格元素：对应各个表格内的元素，比如 thead、tr、td 等，不包裹 table 元素。

无论分为哪一类，他们的盒模型都是基于标准的盒模型，只会有少量的修改。

eg：可以在控制台查看各个元素默认的盒模型。

eg: CSS 的块级元素根据 display 确认的盒模型分类，HTML 的块级元素根据元素分类，二者名字相同，但意义不同。行内元素同理

块级元素

display 属性为以下值时，元素表现为块级元素：

  • block：大部分 HTML 块级元素的盒子。
  • table：是 table 元素的盒子，同 block 不同的是，它的内部是按照表格布局排列。
  • list-item：是 li 元素的盒子，同 block 不同的是，它可以设置列表项的符号（li 元素前面的小点或者数字）。
  • 其他（在更深入的课程介绍）

CSS 块级元素的盒子可以设置盒模型的任意值，内外边距、边框和宽高，并且它们都 独占一行。

行内级元素

CSS 行内级元素的一大特点就是 不会独占一行，超过一行的部分默认会换行显示，对应 HTML 中内联元素的盒模型。

行内块元素

  • inline-block：类似于块级元素，可以设置盒模型的任意值。
  • inline-table：和普通表格的盒子类似。

上述盒模型同 display:block 与 display:table 不同的就是不会独占一行，其他并无区别。

行内元素

  • inline：所有内联元素的默认盒模型。

行内元素大部分尺寸设置都会产生限制，连宽高都无法设置，本节的最后，会介绍行内元素的盒模型限制。

表格元素

表格内元素的盒模型非常特殊，在 HTML 中介绍过，表格内的元素既不是块级元素，也不是内联元素。表格内每种元素几乎都有特定的盒模型。各个元素默认对应的 display 值如下：

元素 默认 diplay 值
caption table-caption
th，td table-cell
tr table-row
tbody table-row-group
thead table-header-group
tfoot table-footer-group
col table-column
colgroup table-column-group

表内的盒模型有以下特点：

  • table-caption 盒子无限制（满足标准盒模型），其他盒模型都没有 margin。
  • table-cell 盒子只是不能设置 margin。
  • table-row 盒子只能设置 border。
  • 剩余盒模型什么都不能设置。

盒模型分类小结

  • display 属性的值非常多，但是不用全部记忆，最常用的盒模型是以下三个：

    • block：块级元素代表
    • inline-block：行内块元素代表
    • inline：行内元素代表

  • 表格内的盒模型仅做了解，通常只会改变单元格的大小和内边距，不会触及到被限制的属性设置，比如很少会在表格中使用 margin，大部分表格内的盒模型在未来都不会遇到。

盒模型计算方式

  • box-sizing

    • content-box 标准盒模型，width 和 height 表示内容宽高
    • border-box 怪异盒模型，width 和 height 表示 border + padding + 内容

文档流

文档流是一个抽象的概念，它表示了元素在页面中的排列方式。

默认情况下的文档流又被称作常规流或正常流。在常规流中，内联元素会自左向右排列，溢出部分则会产生换行，这种布局环境被称作 行内格式化上下文(Inline formatting contexts)简称 IFC ；常规流中的块级元素会自上向下排列，这种布局环境被称作 块格式化上下文(Block formatting contexts)简称 BFC。（表格内之所以有特殊的排列规则，就是因为它内部的文档流不是常规流。）

IFC 和 BFC 描述了一个布局区域和规则，默认情况下，一个 HTML 文件只有一个 BFC 环境，就是 <html> 元素内，而 IFC 就会很多，每一段文字的排列，存在于一个 IFC 内。

BFC 概述

BFC 是块级元素在常规流中布局区域与规则，元素也可以创建一个新的 BFC 环境，一个 BFC 包含了创建它的元素内部的所有内容。比如 <html> 元素创建的 BFC 就包含了它内部的所有内容。

同一个 BFC 内的块级元素满足一下会按照一下方式排列：

  • 块级元素产生的各个盒子依次向下排列，每个盒子独占一行，（嵌套不会产生新的 BFC）；
  • 块级元素之间的盒子在垂直外边距上会出现相互重叠；
  • BFC 内包含创建它的元素内部的所有内容；

IFC 概述

IFC 描述了行内级元素在常规流中的布局区域与规则，行内级元素的每个单词或汉字都会参与到一个 IFC 中，IFC 内的行内级元素会按照一下方式排列：

  • 盒子都会依次按照书写方式排列（默认从左到右）。
  • 当盒子相互排列溢出一行的时候，向下新开一行继续排列。
  • 行包含了本行中所有的内容，行与行之间没有垂直间距。

eg：每行不一定都一样高，某一行可能有一张图片，就会把这行撑得比其他行高。

行内元素的盒子会有些许不同：

  • 行内元素的盒子 width、height 无效，盒子的高度参照字体大小决定不是行高，盒子的长度由字体大小和文本长度决定；
  • margin、border、padding 只在水平方向上正常生效；
  • 垂直方向上：margin 不显示，border 和 padding 显示但是不参与布局计算；
  • 行内元素的盒子会在换行的时候出现断层现象；
