布局进阶

备注: 本节课较难。

包含块

描述：一个元素的框的尺寸和位置的计算有时相对于某个特定的长方形，称为该元素的包含块。
—W3C

元素的布局经常会受到包含块的影响，比如给 margin 设置一个百分比时，参考值就来自元素的包含块。

不管什么元素，都会在依赖他的包含块来布局，超出包含块的部分被称作溢出，通过溢出，可以反向推理出包含块的区域

通过了解包含块可以计算盒模型各部分尺寸的百分值。

确定元素的包含块

一个元素的包含块和他的 position 属性息息相关：

  1. position 的值是 relative 、static 或者 sticky，他的包含块一般是它最近的块元素祖先的内容边界组成。
  2. position 值为 fixed 时，它的包含块由一般由视口创建。
  3. position 值为 absolute 时，包含块由它最近的 position 不是 static 的祖先的内容边界组成。

计算盒模型的百分值

  1. height、top 和 bottom 的百分值参考的是包含块的高度。但是有一个特殊情况：

    • 当包含块的 高度不确定 时（也就是高度是被内容撑起时）并且元素 不是绝对定位时，上面的值的百分比值被计算成 auto。

  2. width、left、right、padding 和 margin 的百分值参考的是包含块的宽度。

行内格式化上下文 IFC

浏览器在为行内级元素布局时，会创建对应的行内格式化上下文 IFC，简要的说，它确定了行内级元素如何布局的规则与区域。

字体大小 font-size

  • font-size 并不是字体实际的大小或范围，而是给字体渲染时提供了一个基础值，字体会根据这个基础值缩放，最终展现出来的字体，往往是要比 font-size 更大的。

  • 因为字体的不同，font-size 表现出来的大小并不是完全相同。

  • 如果没有指定 line-height，那么 line-height 等于字体的通过 font-size 计算后的高度。

  • line-height 决定的不是字体的高度，只是一个用来渲染的行高，字体总是会在 line-height 区域的中间

  • 决定垂直方向上的布局的是 line-height，不是字体的高度。

  • 不要认为 line-height 为 100% 的时候和字体一样高，它往往比字体矮。

介绍两个字体的高度：

  • 字体高度：字体根据 font-size 计算渲染出来的高度，可以通过设置背景色看到。
  • 字体行高：由 line-height 确定，line-height 为 normal 的时候，字体行高和字体高度一样。

IFC 环境中的框

看不见，摸不着的两个框：

  • 行内框

    • 行内框和盒模型并没有直接关系

    • 行内框包裹住了各个行内级元素或匿名行内元素

    • 高度

      • 如果是（匿名）行内元素，行内盒的高度是字体行高
      • 如果是行内块元素，行内盒的大小是其盒模型的总高度

    • 宽度

      • 如果是（匿名）行内元素，那么由 font-size、文本长度、margin、border、padding 水平方向的值决定
      • 如果是行内块元素，那么是其盒模型的总宽度

    • （匿名）行内元素行内盒溢出会被截断

  • 行框

    • 行框和盒模型并没有直接关系
    • 包裹住了所在行中所有行内框
    • 高度：由行内框的排列后的高度决定，并不一定是内部最高的行内框高度
    • 宽度：默认情况下是包含块的宽度，如果受到浮动元素的影响，会减去浮动元素占据的宽度

块格式化上下文 BFC

浏览器在为块级元素布局时，会创建相应的块格式化上下文 BFC，相比 IFC 来说，IFC 是一个或多个内联元素参与创建的，而 BFC 一般是由元素直接创建的。BFC 确定了常规流中块级元素的排列规则和区域。

默认情况下,BFC 内的块级元素排版如下：

  • 块级元素产生的各个盒子依次向下排列，每个盒子独占一行，（嵌套不会产生新的 BFC）；
  • 如果块级元素宽度小于行宽，那么设置 margin-left 和 margin-right 为 auto 的时候，元素会居中。
  • 块级元素之间的盒子在垂直外边距上会出现相互重叠；
  • BFC 内包含创建它的元素内部的所有内容；

创建 BFC

以下方式可以为元素创建 BFC ：

  • <html> 元素
  • overflow 值不为 visible 的块元素
  • 行内块元素：display:inline-block
  • 浮动元素：flaot 不是 none
  • 绝对定位和固定定位元素
  • 其他（其他方式会有较大的副作用，或是我们未学到的知识，又或者兼容性不佳，可以查看 这里 了解所有产生 BFC 的方式）

独占一行

因为 BFC 内块级元素独占一行的特性，块级元素的宽度小于当前行时，其余部分会自动被 margin 填充。如果设置水平方向的 margin 为 auto，这个块级元素就会水平居中。

外边距重叠

在常规流的 同一个 BFC 中，如果 相邻 的块级元素之间，没有间隙、内边距、边框、行框阻挡他们的外边距，且他们 垂直相邻，那么他们的外边距部分会发生重叠。

外边距重叠又叫外边距合并，外边距折叠等等，都是一个意思。

BFC 环境内的外边距重叠有以下三种情况：

  • 父子级
  • 兄弟级
  • 兄弟级中有空的块级元素

围绕外边距重叠的定义，有以下方式解决重叠的问题：

  • 让外边距不直接接触
  • 累加重叠部分
  • 建立新的 BFC 环境

是否一定需要解决外边距合并？

不一定，如果并没有影响到我们的布局，我们并不需要去处理外边距重叠，因为它并不是一个 bug，它只是 BFC 环境中块级元素的布局方式。

内容包含

BFC 内包含创建它的元素内部的所有内容。

  • 外边距合并无法重叠到 BFC 外。
  • 浮动元素会撑起 BFC 区域。

浮动、BFC 与 IFC

  • 同一个 BFC 中

    • 块级元素布局会无视浮动元素，浮动元素会浮动在块级元素上，因为浮动在元素上方，所以无法撑起元素的高度。
    • 浮动会挤压周围 IFC 内的行框，造成文本环绕浮动元素布局。浮动对 IFC 的影响与 HTML 结构无关，可以跨越父级子级兄弟级。
    • 浮动元素对相对定位元素的影响基于原始位置，相对定位偏移后并不会消除或改变浮动的影响。
    • 浮动会撑起 BFC 高度。

  • 不同 BFC 中

    • 浮动元素的影响不会干扰到当前 BFC 区域外部。
    • inline-block 行内块元素虽然会产生 BFC，但是它同时也存在于 IFC 的行盒中，当行内块元素的宽度超过行盒宽度，行内块元素会被向下挤压换行，类似于清除浮动。
    • 如果产生 BFC 的元素本身还是一个块级元素，并且还在常规流中，那么它会紧靠浮动元素独占一行，不会因为浮动元素挤压换行。
