对象原型

原型和原型链

每个引用类型数据都会有一个特殊的内置属性 [[prototype]]，这个属性被称作对象的 原型，原型可能是一个对象或者 null，不会是其他类型。

下面是获取引用类型原型的常用方式：

方式 描述
obj.__proto__ 通过引用类型的内置属性获取，这个是非标准的方式
Object.getPrototypeOf(obj) 通过 Object.getPrototypeOf 函数获取

在旧浏览器中上面原型可能无法获得，上面两种方式都可能获取失败，比如 IE 9 以下。

如果一个对象的原型是对象，那么这个对象就也存在原型，他们都被链式关联到一起，这组原型的链式数据被称为对象的 原型链。

可以通过 .__proto__.__proto__... 这种方式来访问对象的原型链，默认情况下，一个对象的原型指向 Object.prototype 对象，而 Object.prototype 的原型指向 null。

设置对象原型

设置原型有以下方式：

  • 通过设置 __proto__ （非标准，但是老浏览器支持）
  • 通过 Object.setPrototypeOf（标准，但是老浏览器不支持）
  • 通过 Object.create（IE9 以下不支持）
  • 使用 new 关键字

通过 Object.create

使用 Object.create 函数也可以设置对象的原型，它接受一个对象或者 null，返回一个空对象，空对象的原型指向传入的值。Object.create 的参数不止一个，其他参数可以用于设置新生成对象的属性及其属性状态，这部分内容不在本系列课程的讨论范围之内，会在 JS 的深入课程中我会介绍。

使用 Object.create，不同于上一节中的直接设置对象原型，这种方式不会给程序带来性能损耗。

关于 Object.create 的参数，请记住，传入值如果是 null，那么会得到一个原型指向 null 的对象，它不能使用 Object.prototype 上的属性，比如 hasOwnProperty。

总结

  • 每个引用类型的数据都存在一个内置属性 [[prototype]]，它被称作引用类型的 原型。
  • 原型可能是一个对象或者 null，null 不是引用类型，所以没有原型。
  • 引用类型的属性访问会顺着原型链查找。
